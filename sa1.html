Programació i Tractament de Dades

Una introducció al pensament computacional i la resolució de problemes.

1. Fonaments de la Programació

Abans de començar a escriure codi, hem d'aprendre a "pensar" com un ordinador. Això implica donar instruccions molt precises i ordenades per resoldre un problema. Aquest conjunt d'instruccions és el que anomenem un algorisme.

1.1. Estructura Seqüencial i Blocs

Un programa informàtic és com una recepta de cuina. Les instruccions s'executen una darrere l'altra, en ordre, des de la primera fins a l'última. Aquesta és l'estructura seqüencial.

Aquests programes s'organitzen en blocs: grups d'instruccions que tenen una finalitat concreta, com demanar dades a l'usuari, fer un càlcul o mostrar un resultat per pantalla.

1.2. Diagrames de Flux i Pseudocodi

Per dissenyar i entendre els algorismes abans de programar-los, fem servir dues eines principals:

  • Diagrama de flux: És una representació gràfica de l'algorisme. Utilitza símbols estàndard per representar les accions (inici/fi, entrada/sortida de dades, processos, decisions).
  • Pseudocodi: És una descripció de l'algorisme utilitzant un llenguatge natural però estructurat, semblant a un llenguatge de programació. És fàcil d'entendre sense necessitat de conèixer una sintaxi específica.

Exemple de diagrama de flux per calcular la suma de dos nombres.

1.3. Variables i Tipus de Dades

Una variable és com una capsa on guardem informació. Cada capsa té un nom (identificador) i conté un valor que pot canviar durant l'execució del programa. Les dades que guardem poden ser de diferents tipus:

  • Integer (entera): Nombres sencers. Ex: 16, -5, 1000.
  • Float (real): Nombres amb decimals. Ex: 9.81, -0.5.
  • String (cadena de text): Text. S'escriu entre cometes. Ex: "Hola, món!", "Joan".
  • Boolean (booleana): Només pot tenir dos valors: True (veritat) o False (fals). S'utilitza per a condicions.
# Exemple de variables en Python
edat = 16               # Integer
altura = 1.75           # Float
nom = "Maria"           # String
es_major_edat = False   # Boolean

print(f"{nom} té {edat} anys.")

1.4. Operadors

Els operadors ens permeten realitzar operacions amb les variables i els valors.

  • Aritmètics: Suma (+), resta (-), multiplicació (*), divisió (/), mòdul (%, residu de la divisió).
  • De comparació: Igual a (==), diferent de (!=), més gran que (>), més petit que (<), més gran o igual que (>=), més petit o igual que (<=). Retornen un valor booleà (True o False).
  • Lògics: and (i), or (o), not (no). S'utilitzen per combinar expressions booleanes.

2. Estratègies i Estructures Avançades

2.1. Bucles Imbricats

Un bucle imbricat és un bucle dins d'un altre bucle. Són molt útils per treballar amb estructures de dades de dues dimensions, com taules, o per crear patrons.

Imagina que vols dibuixar un quadrat de 4x4 asteriscs. Necessites un bucle "exterior" que controli les files i un bucle "interior" que controli les columnes (els asteriscs de cada fila).

# Dibuixem un quadrat de 4x4
# El bucle exterior s'executa 4 cops (un per cada fila)
for i in range(4):
    # El bucle interior també s'executa 4 cops PER CADA iteració de l'exterior
    for j in range(4):
        # print sense salt de línia
        print("* ", end="")
    # Quan el bucle interior acaba, fem un salt de línia per passar a la següent fila
    print()

# Sortida:
# * * * * # * * * * # * * * * # * * * *

Per cada iteració del bucle exterior, el bucle interior s'executa completament. Això ens permet recórrer tots els elements d'una graella o matriu.

2.2. Funcions Pròpies

Una funció és un bloc de codi reutilitzable que realitza una tasca específica. En lloc de repetir el mateix codi diverses vegades, el podem agrupar en una funció i cridar-la quan la necessitem.

Avantatges d'utilitzar funcions:

  • Reusabilitat: Escrius el codi un cop i el fas servir tantes vegades com vulguis.
  • Llegibilitat: El codi principal queda més net i és més fàcil d'entendre.
  • Manteniment: Si has de corregir un error, només ho has de fer en un lloc: dins de la funció.
# Definim una funció que saluda
# 'nom_usuari' és un paràmetre: una variable que rep un valor quan cridem la funció
def saludar(nom_usuari):
    print(f"Hola, {nom_usuari}! Benvingut/da al curs.")

# Cridem la funció diverses vegades amb diferents arguments
saludar("Anna")
saludar("Pau")

# Definim una funció que retorna un valor
def calcular_area_rectangle(base, altura):
    area = base * altura
    return area # La paraula clau 'return' envia el resultat cap a fora

# Cridem la funció i guardem el resultat en una variable
area_total = calcular_area_rectangle(10, 5)
print(f"L'àrea del rectangle és: {area_total}")

3. Estratègies de Resolució de Problemes

3.1. Divideix i Guanyaràs

És una de les estratègies més importants en programació. La idea és simple: si un problema és massa gran i complex, el dividim en problemes més petits i manejables. Resolem cada petit problema per separat i després combinem les solucions per resoldre el problema original.

Les funcions són l'eina perfecta per aplicar aquesta estratègia. Cada funció s'encarrega de resoldre una part del problema.

Exemple: Volem un programa que calculi la mitjana de les notes d'una llista.

Podem dividir-ho en:

  1. Una funció per sumar totes les notes.
  2. Una funció per comptar quantes notes hi ha.
  3. Combinar els resultats per calcular la mitjana (suma / total).
def sumar_llista(numeros):
    suma = 0
    for num in numeros:
        suma += num
    return suma

def calcular_mitjana(llista_notes):
    # Problema petit 1: sumar les notes
    total_suma = sumar_llista(llista_notes)
    
    # Problema petit 2: comptar quantes notes hi ha
    quantitat_notes = len(llista_notes)
    
    # Combinem les solucions
    mitjana = total_suma / quantitat_notes
    return mitjana

# Programa principal
notes_classe = [8.5, 7.0, 9.2, 6.5, 10.0]
mitjana_final = calcular_mitjana(notes_classe)
print(f"La mitjana de la classe és: {mitjana_final:.2f}")

3.2. Recursivitat

La recursivitat és una tècnica on una funció es crida a si mateixa per resoldre un problema. És una alternativa als bucles per a certs tipus de problemes.

Tota funció recursiva ha de tenir dues parts:

  • Cas base: Una condició que atura la recursivitat. Sense ell, la funció es cridaria a si mateixa infinitament.
  • Cas recursiu: La part de la funció on es torna a cridar a si mateixa, però amb un problema lleugerament més senzill.

Exemple: Calcular el factorial d'un nombre (ex: 5! = 5 * 4 * 3 * 2 * 1).

def factorial(n):
    # Cas base: si n és 1, el factorial és 1. Aquí s'atura.
    if n == 1:
        return 1
    # Cas recursiu: n * factorial(n-1)
    else:
        return n * factorial(n - 1)

# Com funciona per factorial(4)?
# 1. factorial(4) -> 4 * factorial(3)
# 2. factorial(3) -> 3 * factorial(2)
# 3. factorial(2) -> 2 * factorial(1)
# 4. factorial(1) -> retorna 1 (cas base)
# Ara, es reconstrueix el resultat: 2 * 1 = 2 -> 3 * 2 = 6 -> 4 * 6 = 24

resultat = factorial(4)
print(f"El factorial de 4 és: {resultat}") # Sortida: 24

La recursivitat és un concepte molt potent, especialment útil en problemes matemàtics i en l'exploració d'estructures de dades complexes.
